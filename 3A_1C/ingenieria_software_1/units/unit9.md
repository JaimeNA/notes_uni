# Estrega y delivery de producto de software

Generalmente esta etapa se considera aislada del resto del desarrollo, pero hoy eso quedo 
obsoleto. Es mas bien el comienzo de agregar valor del cliente, haciendo que este proceso 
sea un punto estrategico porque permite observar como usan el sistema los clientes 
finales.

## Modelos de entregas

- SaaS(Software as a Service), el proveedor maneja todo.
- On-Premise, el cliente debe manejar el producto.
- Llave en Mano, el proveedor instala todo y luego le cede el control al cliente.
- Multitenant, una unica instancia sirve para multiples clientes.
- Hibrido / Custom Deploy, el resto de instalaciones que no se pueden clasificar.

## Ciclo de puesta en marcha

Independientemente del modelo de entrega, todos tienen:

- Preparacion del entorno
- Despliegue tecnico
- Validacion funcional
- Capacitacion y documentacion
- Handover y monitoreo

> Todo tiene que estar bien explicado para que el cliente pueda seguir manteniendolo.

## Contratos de soporte y ongoing

- Correctivo
- Evolutivo
- Preventivo
- Adaptativo
- Capacitacion y transferencia
- Soporte Ongoing / Success Fee
- Outsourcing de Operacion / Managed Service

## Reflexion final

La entrega marca el inicio de una nueva etapa: la del valor de uso.
Hay que garantizar que ese software evolucione junto al clinete, que se adapte a sus 
procesos y potencie sus resultados. 

> La verdadera entrega no ocurre cuando se instala el fotware, sino cuando empieza a 
transformar la realidad para la cual fue creado.
