Para textos:
- Soundex:  O(N), N longuirud de palabra.
- Methaphone:   No tenemos que implementarlo.
- Levenshtein Distance: O(N*M), sirve para obtener menor distancia.
- Q-Grams: Divide string en grams y va comparando cuantos grams tienen en comun.

Para busqueda exacta:
- Fuerza Bruta o Naive: O(m*N), va y vuelve
- Knuth-Morris-Pratt: O(N+M)

Estructuras lineales:
- Quicksort: O(N*log(N)), pero si el arreglo ya esta ordenado(asc o dec) sera O(N^2)
- Mergesort: O(N*log(N))
- Binary search: O(log(N))

Lucene: 
- FL = sqrt(#frec term en DOCi / #term existentes en DOCi)
- FG = 1 + ln((1+#docs en la coleccion)/(1+#docs que cont query))
- Si es OR, se suman FL*FG para cada termino, si es AND solo se suman los doc que cumplen ambos, pero se suman igual

BigO: 
-  S i = N(N=1)/2 
- MT:
    T(N)= a*T(N/b) + c*N^d

    a < B^d -> O(N^d)
    a = b^d -> O(N^d * log(N))
    a > b^d -> O(N^(log_b(a)))
